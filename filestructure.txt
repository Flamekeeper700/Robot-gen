========================================
UNIVERSAL GEN FILE STRUCTURE LEGEND
========================================

UNIVERSAL GEN/
├── index.html              # Main application entry point
├── filestructure.txt       # Legend file
├── update_configs.js       # Node.js script for fetching/parsing APIs locally
├── package.json            # Node project file for updater script dependencies
│
├── styles/                 # Styling
│   └── styles.css          # Main UI styles
│
├── js/                     # Frontend Application Logic
│   ├── app.js              # UI interactions, state management, event handlers
│   ├── generator.js        # ZIP builder, file generation, context assembly
│   └── parser.js           # JSON loader, config caching, template helpers
│
├── configs/                # Automatically generated API configuration files
│   ├── types.json          # Universal primitive to Java type mappings
│   ├── definitions.json    # API blueprints (classes, methods, parameters)
│   └── templates.json      # Structural skeletons for generated Java files
│
└── .github/
    └── workflows/
        └── update-configs.yml # GitHub Actions automation pipeline

========================================
AUTOMATION WORKFLOW:
========================================
1. GitHub Actions runs `update_configs.js` on a schedule.
2. Script fetches remote API manifests, transforms them, and overwrites `/configs/`.
3. GitHub Actions commits and pushes the new JSON files to the repository.
4. Static frontend (`index.html`) reads the newly updated `/configs/` on load.
